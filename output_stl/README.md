# 딤섬 찜기 클리커 — 생성된 STL (네모 + 둥근 모서리 버전)

`python3 generate_dimsum_clicker.py` 실행으로 재생성. 치수는 스크립트 상단 파라미터로 조정.

## 파일

| 파일 | 내용 | 크기 |
|---|---|---|
| `floor1_battery.stl` | 1층: 520mAh 배터리 (낮은 층) | 44 × 39 × 7.5mm (+턱 1.5) |
| `floor2_esp32.stl` | 2층: ESP32 + 충전모듈 + OLED + USB-C | 44 × 39 × 20.5mm (+턱) |
| `floor3_switch_lid.stl` | 3층: 스위치 스탠드 뚜껑 | 44 × 39 × 10mm |
| `assembly_preview.stl` | 조립 확인용 (출력 금지) | 전체 44 × 39 × 38mm |

## 구조

- **44 × 39mm, 모서리 R8** 둥근 네모 — 네모라서 배터리(36.5×28.5)가 대각선 걱정 없이 들어가 원형(Ø54)보다 작아짐
- **결합**: 스크린샷 방식 그대로 — 아래층 상단 안쪽 턱(ridge, 1.2폭 × 1.5높이) 위로 위층 벽 스커트가 덮여 겉면이 플러시하게 맞물림 (여유 0.2/측)
- **1층 (h7.5)**: 배터리 얇은 고정 테두리(37.2×29.2). 배선은 2층 바닥판 구멍으로 위로
- **2층 (h20.5)**:
  - ESP32 포켓 18.4×24.4 세로 배치 (서쪽), 충전모듈 포켓 19.4×14.4 (동쪽, USB가 벽 방향)
  - 동쪽 벽: `usb_c_hole.stl` 형상(나팔부 60% 압축) USB-C 관통 구멍
  - 북쪽 벽: OLED(15×16×2.4) 세로 슬롯(위에서 삽입) + 핀 배선 통로 + 13.5×8 디스플레이 창
  - 바닥판 배선 구멍 8×4 @ (4, -12.5) → 1층 배터리 선 통과
- **3층 (h10, 뚜껑)**: 상판(3.2t)에 `keyboard_switch_stand.stl` 바닥 실측 구멍 — 중앙 7×7 관통(기둥+배선), 다리 2.3² ×2, 안착 홈 14.9×15.3×0.6

## 출력 팁

- 1·2층: 그대로 출력 / **3층은 뒤집어서** (상판이 베드에)
- rabbet 결합부 천장(폭 1.6mm)은 짧은 브릿지 — 서포트 불필요
- 공차: 포켓 0.4, 결합 0.2/측, 배터리 0.7 — 프린터에 맞게 스크립트에서 조정
