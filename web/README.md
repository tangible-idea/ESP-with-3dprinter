# 🥟 딤섬 클리커 웹 컨피규레이터

브라우저에서 슬라이더로 케이스를 실시간 재설계하고 층별 STL을 바로 다운로드합니다.

## 실행

```bash
# 프로젝트 루트에서 (stl_files/ 를 읽어야 하므로 반드시 루트에서)
python3 -m http.server 8123
```

브라우저에서 <http://localhost:8123/web/> 접속.

## 기능

- **모양**: 둥근 네모 / **완전 원형(딤섬 찜기)** — 원형은 W가 지름이 되고(기본 Ø54, 배터리 대각선 46.3mm 때문에 하한) D·모서리는 비활성. 원형에서는 충전모듈이 항상 중앙(Y 고정)이고 동쪽 벽에 폭 18mm 플랫 USB 패드가 생김(my_designs/rounded.png 참조). OLED 안착면은 곡률에 맞춰 자동 계산됨
- **외곽 디자인**: 가로/세로/모서리 둥글기/벽 두께/결합 유격. 결합 유격(기본 0.08mm)을 줄일수록 층이 꽉 끼움
- **V형 결합부** (기본 켜짐): 층간 홈·턱을 58° 계단 단면으로 만들어 위층 밑면에 서포트가 필요 없음 + 조립 시 자가 정렬. 끄면 기존 사각 단면
- **설정 자동 저장**: 모든 값이 localStorage에 저장되어 리로드해도 유지, '설정 초기화' 버튼으로 기본값 복원
- **층 높이**: 1·2·3층 + 스위치 보스 높이/스탠드 매립 깊이
- **부품 배치**: ESP32 위치·회전, 충전모듈 Y(USB는 항상 동쪽 벽), OLED 종류(0.49" 15×16 / 0.96" 26×26)와 벽 선택(서쪽 기본 = USB 반대), 배선구멍 위치·방향(가로/세로 14×5 슬롯 — 배터리 +/− 두 가닥이 함께 통과)
- **0.96" OLED**: 표시영역 22.74×11.86에 맞는 큰 창 + 모서리 4홀(22×22mm)에 맞는 Ø1.8 위치결정 핀 4개가 포켓 안에 자동 생성. 모듈이 26mm로 커서 타워가 높아지므로 2·3층 높이 경고를 확인할 것
- **OLED 타워 결합**: OLED는 타워 뒤(내부)에서 통째로 밀어 넣음 — 뒷면 완전 개방(15.5×16.3), 앞은 벽+디스플레이 창이 잡아줌. OLED 소켓은 2층 높이와 무관하게 항상 21.2mm 타워로 서고, 2층 벽보다 높으면 3층 뚜껑에 타워 모양 노치가 자동으로 커팅되어 끼워짐(조립 키 역할). 덕분에 2층을 16mm까지 낮출 수 있음
- **분해 ⟷ 조립** 슬라이더 + 애니메이션, 부품 고스트 표시, 케이스 반투명(X-ray)
- **조립 간섭 자동 검사**: 리빌드마다 층간 교집합 부피를 계산해 하단에 "조립 간섭 없음 ✓" 표시
- **STL 내보내기**: 층별 다운로드, 3층은 출력용 뒤집기 옵션 (크롬이 연속 다운로드를 차단하면 주소창의 다운로드 아이콘에서 허용)
- 배터리/부품이 안 들어가는 조합이면 하단에 경고 표시

## 구현

- CSG 엔진: **manifold WASM** (`vendor/manifold.wasm`) — 파이썬 생성기(`generate_dimsum_clicker.py`)와 동일 엔진이라 출력 STL이 항상 watertight
- 3층 스위치 자리는 `stl_files/keyboard_switch_stand.stl` 실물 메시를 3.5% 팽창시켜 그대로 boolean 절삭 → 스탠드가 보스에 자연스럽게 매립
- USB 구멍도 `stl_files/usb_c_hole.stl` 실물 형상 사용 (얇은 벽에 맞게 나팔부 60% 압축)
- 모든 라이브러리는 `vendor/`에 로컬 포함 — 오프라인 동작
